# Sequential Deck Review

## Purpose

デッキ全体を一気に直すのではなく、章ごとに順番に見直して品質を上げる。

## Inputs

- HTMLデッキ
- CSS
- スライド構成
- 原稿

## Outputs

- 対象章の修正版HTML/CSS
- レビュー観点メモ

## Procedure

1. 対象範囲を 2〜4ページ程度に絞る。
2. 情報設計、改行、余白、視線誘導、色の使い方を確認する。
3. 直すべき問題を構造、文言、装飾に分ける。
4. 問題の大きい順に修正する。
5. 次の範囲へ進む前に、対象範囲だけで見たときに成立しているか確認する。

## Quality Checks

- 章の入口と出口が自然か
- 強調すべき要素が視覚で伝わるか
- 読みにくい改行や詰まりがないか
- 他ページとスタイル文法が揃っているか

## Failure Cases

- 一度に全部直して、どこが良くなったか分からなくなる
- 見た目だけ直して構造が弱いままになる

## Notes

- 修正対象を限定して、一段ずつ詰める。
