# Page Image Benchmark Generator

## Purpose

各スライドを1ページずつ image 生成し、HTML再現前の個別ベンチマークとして保存する。

## Inputs

- スライド構成
- 各ページの見出しと本文
- デザイン基準となる一覧ボード

## Outputs

- 各ページの生成画像
- ページ別プロンプトメモ
- 採用画像の保存先一覧

## Procedure

1. ページごとに役割を1文で定義する。
2. タイトル、本文、図、装飾の優先順位を整理する。
3. image生成は1ページ単位で行う。
4. 生成後、HTML化時に再現すべき要素をページ別にメモする。
5. 採用画像をプロジェクト配下へコピーする。

## Quality Checks

- 一覧ボードとトーンが揃っているか
- ページ単体で見たときに情報の重心が明確か
- 文字量と余白のバランスが破綻していないか

## Failure Cases

- 文字量を詰め込みすぎて読みにくくなる
- 図とテキストの主従が逆転する
- 同じ資料なのにページごとに別トーンになる

## Notes

- HTML再現を前提に、文字焼き込みのしすぎは避ける。
