# Component Layout Planning

## Purpose

スライド全体を先に生成せず、HTML再構築しやすい単位にレイアウトとコンポーネントを分解する。

## Inputs

- 参考スライド画像
- スライド構成メモ
- コピー原稿

## Outputs

- `design/component_layout_spec.md`
- `design/component_manifest.json`
- 必要に応じて `design/component_prompt_map.md`

## Procedure

1. スライドをページ単位ではなく、レイアウト単位で分類する。
2. 1カラム、2カラム、カードグリッド、ヒーロー、図解などの骨格を定義する。
3. 各スライドをタイトル、本文、カード、図、装飾、ロゴなどの部品に分解する。
4. テキストで持つ部品と画像生成対象の部品を分ける。
5. HTMLで安定配置できる部品サイズと比率を決める。

## Quality Checks

- レイアウト定義が曖昧な言葉で終わっていないか
- テキスト部品と画像部品が混線していないか
- 同一デザインのカードやパネルが再利用可能な形で定義されているか
- HTMLで固定しやすい寸法ルールになっているか

## Failure Cases

- スライド全体をそのまま1コンポーネント扱いしてしまう
- 装飾と意味要素を分けずに定義してしまう
- 後からテキスト差し替えできない設計にしてしまう

## Notes

- 最初にレイアウトを固定しないと、その後のimage生成が毎回ぶれる。
