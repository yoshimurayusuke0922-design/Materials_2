# Image2 Benchmark Pack Builder

## Purpose

HTML実装前に、資料全体のデザイン基準として使う image2 ベンチマーク案を作る。

## Inputs

- スライド構成
- 原稿
- デザイン方針
- 参考資料のトーン

## Outputs

- 一覧ボード画像
- 個別スライド画像
- 生成方針メモ
- 採用候補メモ

## Procedure

1. 先にデッキ全体のデザイントーンを文章で固定する。
2. 全13ページの役割を短く整理する。
3. 1枚の一覧ボードを生成して、全体トーンを確認する。
4. その後、個別ページを順に生成する。
5. HTML再現時に使うため、どこが文字、どこが装飾、どこが図かをメモする。

## Quality Checks

- 全ページのトーンが揃っているか
- 三幸資料ベースの赤白構成が維持されているか
- 洗練方向に寄っているか
- 文字量とレイアウトの釣り合いが取れているか

## Failure Cases

- 1ページ単位の出来は良くても全体でバラつく
- 一覧ボードだけで満足して個別詰めに進まない
- 画像に本文を焼き込みすぎてHTML再現性が落ちる

## Notes

- ベンチマークは完成品ではなく、最終HTMLの視覚基準として使う。
