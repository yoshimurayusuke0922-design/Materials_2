# Demo Placeholder Builder

## Purpose

スクリーンショット未投入のデモ画面ページを、後から差し替えやすい固定フレームで構築する。

## Inputs

- デモ画面ページごとの要件
- ラベルとキャプション
- 画面レイアウト方針

## Outputs

- プレースホルダーHTML
- プレースホルダーCSS
- 差し替え運用メモ

## Procedure

1. 必要な画面数を決める。
2. 各画面枠の比率を固定する。
3. ラベルと説明をHTMLで置く。
4. 実スクリーンショット投入時に崩れない寸法にする。

## Quality Checks

- 枠サイズが安定しているか
- ラベルと枠の対応が分かるか
- ただの空箱に見えすぎないか

## Failure Cases

- スクリーンショット投入時に高さが変わる
- キャプションが多すぎて主役が逆転する

## Notes

- UIらしさは補助。主役は後から入る画面。
