# quality_review

## Purpose

生成 HTML を元画像と比較し、欠落、崩れ、改善点を記録する。

## Inputs

- `outputs/04_html_drafts/`
- `outputs/05_final/`
- 解析結果と元画像情報

## Outputs

- `outputs/06_review/comparison_notes.md`
- `outputs/06_review/issues.md`
- `outputs/06_review/improvement_plan.md`
- `outputs/06_review/conversion_report.md`

## Procedure

1. 元画像と HTML を比較する。
2. 情報欠落、見た目の差、改善余地を列挙する。
3. 未解決点と次の改善案を分けて記録する。
4. 最終的な変換レポートをまとめる。
5. レイアウト修正後は、見た目だけでなく、指定したCSSが実際の対象要素に当たっているかを確認する。
6. 中央揃え、右寄せ、リスト配置、強調文字、画像余白削減は、スクリーンショット上の見え方とDOM/CSSの対象要素を両方確認する。

## Quality Checks

- 情報が抜けていない
- HTML がブラウザで表示できる
- レビュー記録が後から読める
- 印刷プレビューまたはPDF出力時に、見出しの不自然な改行がない。括弧付き語句、助詞、短い接尾語だけが単独行になっていない
- アイコンや図版は、CSS上の寸法ではなく実際に見える大きさで確認する。透明余白のある画像は小さく見えやすいため、カードの主題を支える十分なサイズになっているか見る
- 日本語の本文で、単語・熟語・送り仮名・「について」「できます」「いたします」などが孤立していない
- イラスト周辺に不要な余白がある場合、画像そのものではなく画像コンテナ、grid列幅、gap、paddingが原因になっていないか確認する
- 左デモ領域 + 右カード群のスライドでは、右カード群を親gridの列幅・配置で寄せ、デモ領域が十分に広がっているか確認する
- 中央揃え指定は、本文段落、強調ボックス、箇条書きそれぞれに期待通り効いているか確認する。箇条書きはブロック全体が中央、内部文字が左揃えになっているか見る
- 変更対象外スライドに、共通CSS変更の副作用が出ていない

## Failure Cases

- 見た目の感想だけで終わる
- 何を再現し、何を再設計したかが分からない
- Chrome印刷プレビューでだけ発生する折り返しやサイズ感の崩れを見落とす
- CSSを追加したのに対象要素へ当たっていない状態を見落とす
- 親要素の中央揃えだけを確認して、子要素の左寄せ指定に負けていることを見落とす
- スクリーンショット上の本文、強調ボックス、箇条書きの揃いを個別に確認しない
- 透明余白のあるイラストを見た目ではなくCSS上の数値だけで判断する

## Notes

- 再現できなかった点は隠さず記録する
