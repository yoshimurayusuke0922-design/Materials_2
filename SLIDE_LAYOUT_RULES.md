# Slide Layout Rules

## 1. レイアウト修正の基本方針

- 見た目だけの局所修正をしない。
- まず親コンテナ構造を確認する。
- grid、flex、gap、padding で整える。
- `absolute` や個別 margin で場当たり的に直さない。

## 2. カード内レイアウトの考え方

- イラストと見出しを横並びにしたい場合は、カード上部に header コンテナを作る。
- header 内は `display: flex; align-items: center; gap: ...;` で管理する。
- 本文エリアは header とは分ける。
- 装飾線より下の本文は、原則既存構造を維持する。

## 3. テキスト改行ルール

- 日本語の熟語を途中で分断しない。
- 意味の切れ目で改行する。
- 1文字だけ次の行に残る状態を避ける。
- 必要な場合は、意味の切れ目に `<br>` を入れる。

## 4. 画像サイズ調整ルール

- 画像だけを無理やり大きくしない。
- 親コンテナ内に収める。
- `max-width: 100%; height: auto; object-fit: contain;` を基本にする。
- 同じ種類のカードでは画像サイズを統一する。

## 5. 変更前後の確認項目

- 修正対象以外のスライドが崩れていないか。
- 文字が不自然に改行されていないか。
- 画像がはみ出していないか。
- テキストと画像が重なっていないか。
- PDF化・スクリーンショット化しても表示が崩れていないか。

## 6. 装飾線・背景グラフィックの扱い

- 不要な灰色の区切り線や背景装飾は削除対象とする。
- 単なる飾りで情報価値がない場合は消す。
- 消す際は、本文やコンテンツ位置に影響を与えない。
- 線の原因が border、box-shadow、背景画像、疑似要素のどれかを確認してから、対象スライドに閉じて無効化する。

## 7. 右寄せ・幅配分の考え方

- 「もっと右に寄せる」は、要素単体を無理に動かすのではなく、親の2カラムまたは複数カラム配分を見直す。
- 左右比率、grid の列幅、`justify-content`、gap を調整して対応する。
- デモエリアを広くしたい場合は、右カラム群だけを押し込まず、全体レイアウトの比率を見直す。

## 8. 強調テキストの扱い

- 一文の中で一部だけを強調する場合は、該当部分を別 `span` に分ける。
- 数字の強調はフォントサイズ、太字、行高を調整する。
- 一行維持よりも、可読性とレイアウト崩れの防止を優先する。

## 9. 画像サイズ調整の追加ルール

- 画像サイズを上げるときは、画像単体ではなくヘッダーやカード全体とのバランスで調整する。
- 画像とテキストが重ならないことを確認する。
- 同じ種類のカードでは、できるだけルールを揃える。
- 左右で役割が異なるカードは、親カードにスコープして個別の基準サイズを持たせる。

## 10. 画像・イラスト余白の扱い

- イラスト周辺に余白が大きい場合、まず画像サイズではなく画像コンテナの余白を確認する。
- `padding`、`margin`、`gap`、固定幅を見直す。
- 透明背景のイラストは、カード内で重ならない範囲なら多少テキスト側に近づいてもよい。
- イラスト余白を削る目的は、テキスト領域を広げて不自然な改行を防ぐこと。
- 画像単体を拡大・縮小するだけで解決しようとしない。

## 11. 右カード群とデモエリアの幅配分

- 左にデモ画面、右にカード群があるスライドでは、親を2カラムまたは3カラム構造として管理する。
- 右カード群を右に寄せたい場合は、`justify-self: end`、`justify-content`、`grid-template-columns` を使う。
- `left` や `transform` で無理やり右に動かさない。
- デモ画面エリアを広げたい場合は、右カード群の不要な余白を削ってから、全体の列幅を調整する。

## 12. 日本語改行ルールの追加

- 単語の途中で改行しない。
- 熟語の途中で改行しない。
- 送り仮名だけを次行に残さない。
- 1文字だけ行頭・行末に残さない。
- 「について」「できます」「いたします」など、助詞・補助動詞だけが孤立しないようにする。
- 必要なら意味の切れ目に `<br>` を入れる。
- ただし、`<br>` を入れすぎて行数を増やしすぎない。

## 13. 中央揃えが反映されない場合の確認ルール

- `text-align: center` は親要素ではなく、実際にテキストを持つ本文要素に当たっているか確認する。
- 子要素側に `text-align: left` が残っていると中央揃えにならないため、本文段落と箇条書きを分けて指定する。
- 箇条書きは、ブロック全体を中央配置し、内部テキストは左揃えにする。
- CSS の読み込み順、セレクタ優先度、対象クラスのズレを確認する。
- 対象スライド以外に影響しないよう、必ず `.slide-12` や実際のスライド専用クラスでスコープを切る。
